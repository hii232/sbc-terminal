# SBC Terminal

Owner-earnings stock terminal for a fixed 126-company official universe. The app is static HTML/CSS/JS and is published with GitHub Pages.

## Live Link

Production: https://hii232.github.io/sbc-terminal/index.html

## Universe

The terminal intentionally keeps the SEC/SBC-ranked official universe at exactly 126 stocks. The source of truth is `data/universe.json` and `universe.js`, generated by:

```bash
python scripts/build_universe.py
```

Builds and tests fail if the official universe is not exactly 126, duplicated, missing CIK mappings, or contains an unapproved ticker. TSM is part of the current official universe; FLUT is not.

## Data Priority

Financial data follows this order:

1. SEC filing facts from `sec.js` and `data/companies/*.json`
2. Company investor-relations filing evidence when manually added
3. Secondary provider checks
4. Clearly labelled estimates
5. Missing

Missing values must stay missing. They are not converted to zero. If SBC, buybacks, net income, shares, or another required field is unavailable, owner earnings, adjusted valuation, quality score, and main ranking are blocked.

SEC annual facts are aligned by exact `periodEnd`, not fiscalYear labels. The runtime builds annual rows from SEC history so revenue, net income, SBC, diluted shares, operating cash flow and capex belong to the same filing period. Free cash flow is calculated from aligned SEC operating cash flow minus aligned SEC capex.

Conflict labels are separated: true conflicts, period mismatches, definition mismatches, unit mismatches, stale/missing facts and missing SEC facts are tracked separately. The red source-conflict badge is reserved for same-period, same-definition disagreements.

Finnhub and FMP are optional browser keys. Finnhub is used for quotes, news, analyst targets and insider data. FMP is stored only as fallback/second-source information and must never overwrite valid SEC-backed financial arrays.

## Market / Business Model

Current SBC model version: `SBC_MODEL_VERSION = "4.0.0"`.
Current market/business model version: `MARKET_TERMINAL_VERSION = "4.1.0"`.

The main company view no longer uses Clean/Middle/High/Tragic as the whole-stock opinion. Those labels remain inside the SBC X-Ray only. The top-level dashboard answers six separate questions:

1. Business Quality
2. Growth and Execution
3. Market Reward
4. Shareholder Economics
5. Valuation
6. Data Confidence

Data Confidence is a separate trust gate. It is shown beside the score, but it is not added to Long-Term View or Market Reward View.

The valuation formula is direct:

```text
Owner earnings per share = adjusted owner earnings / diluted weighted-average shares
Owner-earnings P/E = current price / owner earnings per share
```

Retention percentage is kept as an explanation of the SBC haircut, not as the main P/E shortcut.

The terminal includes two main combined views:

- Long-Term Investment View: quality, growth, valuation, shareholder economics, and market reward.
- Market Reward View: estimate revisions, relative strength, acceleration, surprise/guidance placeholders, sector strength, and post-earnings reaction when point-in-time history exists.

Analyst estimate history lives under `data/estimates/history/*.json` and is generated into `estimates.js`. The GitHub Action `.github/workflows/estimate-history.yml` runs daily on weekdays. Without an `FMP_API_KEY` secret, histories remain explicitly empty and revision fields stay unavailable rather than being invented.

Legacy SBC ranking requires data confidence of at least 80. Below 80, the app shows:

```text
Not ranked - more filing data is needed
```


GitHub Actions also checks exact universe size, required SEC source files, calculation regressions, browser smoke coverage, the golden audit, and simple secret patterns.

## Earnings Command Center

The Earnings Command Center is the terminal's earnings-season cockpit:

- **Beat/miss tape** — every universe company that reported in the last 45 days with actual vs consensus EPS/revenue and the surprise. With a free Finnhub key connected, the tape re-checks itself automatically every few minutes (fast-lane polling during pre-open and after-close report windows) and flashes the moment new actuals land. Without a key, the bundled pipeline stamps results every weekday morning.
- **Beat Odds** — a transparent 0-100 composite on every upcoming report: beat track record (28), estimate-revision momentum (24), pre-report tape (14), sector read-through (14), macro regime (10), and expectation bar (10). Peers' results flow into sector-mates' odds automatically (micro events); the macro regime is computed live from the SPY/sector tape (macro events) — never a hardcoded snapshot. Missing inputs lower coverage instead of being scored as neutral. It is a research signal, not a probability and not investment advice.
- **Season scorecard + sector read-through** — beat rates, average surprise, and which sectors are clearing the bar this season.

The data layer is `earnings.js` / `data/earnings_intel.json`, generated by `scripts/collect_earnings.py` (keyless Yahoo quoteSummary: earningsHistory, calendarEvents, earningsTrend) inside the daily data-refresh workflow. The collector stamps `reportedOn` the first morning a new fiscal quarter appears, building a real report-date ledger over time; first-ever ingests never fake report dates for old quarters.

## The Edge Layer

Evaluations are the encyclopedia; the edge lives in four dedicated systems:

1. **What Changed (signals feed)** — `scripts/build_signals.js` diffs every tracked input daily: score inflections and threshold crossings, Direction Edge flips, revision-tape sign flips, consensus-drift inflections, Beat Odds regime entries, fresh beats/misses, and same-day SEC filing diffs (growth acceleration/deceleration, SBC burden, share-count turns). Materiality-ranked, 21-day rolling ledger (`signals.js` / `data/signals_history.json`). Deltas only — nothing backfilled.
2. **Drift Board (PEAD)** — post-earnings-drift setups on the Earnings Command Center: surprise size + revenue confirmation + post-report revisions + tape confirmation, decaying over the documented ~60-day drift window.
3. **Filing diffs** — the day a new SEC accession is ingested, the signals engine compares the filing's facts against the prior period and emits the change, at the feed's highest impact weight.
4. **Signal calibration** — daily snapshots record every displayed signal (verdict call, Direction Edge, Beat Odds, Market Reward); Track Record grades each bucket against 4- and 12-week forward returns with hit rates. Verdicts are withheld under 20 observations; non-predictive signals get deleted, not defended.

## The 13-View Terminal

The app was deliberately cut down to thirteen views an investor actually uses — Home, What Changed (signals), Earnings Command Center, Daily Review, Direction Edge, Sectors, Rankings, Screener, Compare, Portfolio, Thesis Journal, Track Record, and Data Audit — with seven tabs per ticker (Overview, Quality, SBC X-Ray, Graham Value, Financials, Earnings, News).

Removed as noise or duplication (2026-07): the Stocktwits Social Buzz desk, the hardcoded-snapshot Inflation Desk, the Narratives view (including Polymarket odds), the Options Desk trade tickets, the standalone Owner-Earnings P/E / Graham / Quality-Map / Triggers / Tech-Desk leaderboards (Rankings + Screener cover them; the underlying engines still power ranking columns and per-ticker tabs), and the per-ticker Expectations/Alerts/Framework tabs. Bundled per-name IV/RV/put-call data remains an input to Direction Edge.

## Phone Use

Open the production link on your phone and use Add to Home Screen. Because it is hosted on GitHub Pages, it works on mobile data and does not require the same Wi-Fi as your computer.

## Known Limits

This is a research terminal, not investment advice. Some supplemental cash-flow/capex comparisons remain visible as source conflicts when provider presentation differs from SEC facts. Full manual filing review is tracked in `AUDIT.md`.

Full historical factor backtesting is intentionally blocked until point-in-time fundamentals, estimate revisions, guidance, surprise history and earnings-reaction snapshots accumulate. The current backtest report includes a no-lookahead price-momentum pilot only and refuses to treat current snapshot fundamentals as historical evidence.
